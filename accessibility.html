การเข้าถึงเว็บไซต์ (Accessibility Statement)

ปรับปรุงล่าสุด: 3 กรกฎาคม 2569 · อ้างอิงมาตรฐาน Web Content Accessibility Guidelines (WCAG) 2.1 ระดับ AA

บริษัท ซีเอส.ไซน์ แอนด์ โปรดักส์ จำกัด ("บริษัท", "เรา") มุ่งมั่นให้เว็บไซต์ cssign.co.th สามารถเข้าถึงและใช้งานได้โดยผู้ใช้ทุกกลุ่ม รวมถึงผู้ใช้ที่มีความบกพร่องทางการมองเห็น การได้ยิน การเคลื่อนไหว หรือการรับรู้ เอกสารฉบับนี้อธิบายเป้าหมายด้านการเข้าถึง (accessibility) ของเรา สิ่งที่ดำเนินการแล้ว ข้อจำกัดที่ทราบอยู่ และช่องทางแจ้งปัญหาการใช้งาน

สารบัญ
  1. มาตรฐานที่ยึดถือ
  2. สถานะการปฏิบัติตามมาตรฐาน
  3. ฟีเจอร์การเข้าถึงที่รองรับแล้ว
  4. ข้อจำกัดที่ทราบและแผนแก้ไข
  5. เทคโนโลยีที่ใช้ในการพัฒนา
  6. วิธีการประเมิน
  7. แจ้งปัญหาการเข้าถึง

1. มาตรฐานที่ยึดถือ

เรากำหนดเป้าหมายให้เว็บไซต์สอดคล้องกับ Web Content Accessibility Guidelines (WCAG) 2.1 ระดับ AA ซึ่งเผยแพร่โดย W3C — มาตรฐานเดียวกับที่หน่วยงานราชการไทยอ้างอิงในระเบียบว่าด้วยเว็บไซต์หน่วยงานภาครัฐ และเป็นมาตรฐานที่หน่วยงานรัฐ/รัฐวิสาหกิจ ตลอดจนบริษัทข้ามชาติจำนวนมากกำหนดไว้ในเงื่อนไขการจัดซื้อจัดจ้าง (procurement) ของตนเอง

2. สถานะการปฏิบัติตามมาตรฐาน

เว็บไซต์นี้อยู่ในสถานะ "สอดคล้องบางส่วน" (Partially Conformant) กับ WCAG 2.1 ระดับ AA กล่าวคือ บางส่วนของเนื้อหายังไม่สอดคล้องกับมาตรฐานอย่างสมบูรณ์ตามรายละเอียดในหัวข้อ "ข้อจำกัดที่ทราบ" ด้านล่าง เรากำลังปรับปรุงอย่างต่อเนื่องและจะอัปเดตสถานะนี้เมื่อมีความคืบหน้า

3. ฟีเจอร์การเข้าถึงที่รองรับแล้ว

  • ลิงก์ข้ามไปเนื้อหาหลัก (Skip to content): ทุกหน้ามีลิงก์ซ่อนที่ปรากฏเมื่อกด Tab เป็นปุ่มแรก ให้ผู้ใช้คีย์บอร์ด/สกรีนรีดเดอร์ข้ามเมนูนำทางไปยังเนื้อหาหลักได้ทันที
  • โครงสร้าง HTML เชิงความหมาย (semantic markup): ใช้แท็ก <header>, <nav>, <main>, <footer> และลำดับหัวข้อ (heading) ที่เป็นลำดับชั้นถูกต้อง เพื่อให้สกรีนรีดเดอร์อ่านโครงสร้างหน้าได้ถูกต้อง
  • ป้ายกำกับสำหรับผู้ช่วยเทคโนโลยี (ARIA labels): ปุ่มไอคอน เช่น เมนูมือถือ ช่องค้นหา และวิดเจ็ตแชท มีข้อความ aria-label กำกับความหมาย ไม่ใช่แค่ไอคอนเปล่า
  • คำอธิบายภาพ (alt text): รูปภาพประกอบเนื้อหา เช่น โลโก้และรูปผลงาน มีข้อความ alt อธิบายเนื้อหาของภาพ
  • นำทางด้วยคีย์บอร์ดได้: เมนู ปุ่ม และลิงก์ทั้งหมดสามารถเข้าถึงและใช้งานได้ด้วยคีย์บอร์ดอย่างเดียวโดยไม่ต้องใช้เมาส์
  • หยุดเนื้อหาที่เลื่อนอัตโนมัติได้: แถบรีวิวลูกค้า (testimonial carousel) ที่เลื่อนอัตโนมัติจะหยุดเลื่อนทันทีที่มีการโฟกัสด้วยคีย์บอร์ดหรือชี้เมาส์ค้างไว้ ตามข้อกำหนด WCAG 2.2.2 (Pause, Stop, Hide)
  • ระบุภาษาของหน้า: ทุกหน้าระบุแอตทริบิวต์ lang ที่ถูกต้อง (th สำหรับหน้าภาษาไทย, en สำหรับหน้าภาษาอังกฤษ) เพื่อให้สกรีนรีดเดอร์ออกเสียงถูกต้อง
  • คอนทราสต์สี: โทนสีหลักของเว็บไซต์ (น้ำเงินเข้ม #0B5A96 บนพื้นขาว/เข้ม) ออกแบบให้มีค่าคอนทราสต์ที่อ่านง่าย สำหรับข้อความเนื้อหาหลัก
  • ออกแบบตอบสนอง (responsive) และรองรับการซูม: หน้าเว็บปรับตามขนาดหน้าจอและรองรับการซูมของเบราว์เซอร์โดยไม่ทำให้เนื้อหาซ้อนทับหรือหายไป

4. ข้อจำกัดที่ทราบและแผนแก้ไข

แม้จะพยายามให้ครอบคลุมที่สุด แต่บางส่วนของเว็บไซต์ยังมีข้อจำกัดที่เรารับทราบและอยู่ระหว่างวางแผนแก้ไข ได้แก่

  • เอกสารประกอบมาตรฐาน: หากต้องการเอกสารประกอบมาตรฐานในรูปแบบที่เข้าถึงได้ กรุณาติดต่อเราตามช่องทางด้านล่าง เรายินดีจัดส่งเวอร์ชันที่เหมาะสมให้
  • การตรวจสอบคอนทราสต์แบบเจาะจง: ยังไม่ได้ตรวจสอบค่าคอนทราสต์ของทุกองค์ประกอบ (เช่น ข้อความบนพื้นหลังภาพถ่ายในบางส่วนของหน้าผลงาน) ด้วยเครื่องมืออัตโนมัติอย่างครบถ้วน 100%
  • วิดเจ็ตแผนที่และแชทของบุคคลที่สาม: องค์ประกอบที่ฝังจากผู้ให้บริการภายนอก (เช่น QR code, แผนที่) อาจมีระดับการรองรับการเข้าถึงที่ขึ้นอยู่กับผู้ให้บริการนั้น ๆ ซึ่งอยู่นอกเหนือการควบคุมโดยตรงของเรา
  • ยังไม่ได้ผ่านการตรวจสอบโดยผู้เชี่ยวชาญภายนอก: การประเมินปัจจุบันอ้างอิงการตรวจสอบภายในและเครื่องมืออัตโนมัติเป็นหลัก (ดูหัวข้อถัดไป) ยังไม่ได้ผ่านการตรวจสอบโดยผู้ใช้จริงที่ใช้เทคโนโลยีสิ่งอำนวยความสะดวก (assistive technology) หรือผู้ตรวจสอบภายนอกที่ได้รับการรับรอง

5. เทคโนโลยีที่ใช้ในการพัฒนา

เว็บไซต์นี้พัฒนาด้วย HTML, CSS และ JavaScript มาตรฐาน (ไม่มีเฟรมเวิร์กที่จำกัดการเข้าถึง) การทำงานของเว็บไซต์ขึ้นอยู่กับฟีเจอร์เหล่านี้ในการแสดงผลอย่างสมบูรณ์ ความเข้ากันได้จะดีที่สุดกับเบราว์เซอร์และเทคโนโลยีสิ่งอำนวยความสะดวกรุ่นที่ยังได้รับการดูแลรักษาอยู่ (เช่น Chrome, Edge, Safari, Firefox เวอร์ชันล่าสุด ร่วมกับสกรีนรีดเดอร์ เช่น NVDA, JAWS, VoiceOver)

6. วิธีการประเมิน

CS.SIGN ประเมินการเข้าถึงของเว็บไซต์นี้ด้วยวิธีการดังนี้

  • การตรวจสอบตนเอง (self-evaluation) โดยทีมพัฒนา อ้างอิงเกณฑ์ WCAG 2.1 ระดับ AA
  • การทดสอบนำทางด้วยคีย์บอร์ดเพียงอย่างเดียว (ไม่ใช้เมาส์) ในหน้าหลัก

7. แจ้งปัญหาการเข้าถึง

เรายินดีรับฟังหากท่านพบอุปสรรคในการเข้าถึงเนื้อหาส่วนใดของเว็บไซต์ กรุณาแจ้งเราพร้อมระบุ URL ของหน้าที่พบปัญหาและอุปกรณ์/เทคโนโลยีสิ่งอำนวยความสะดวกที่ใช้งาน เพื่อให้เราสามารถตรวจสอบและแก้ไขได้ตรงจุด
